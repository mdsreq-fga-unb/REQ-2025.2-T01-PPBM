## Resumo das Reuniões

### Sprint 2 Planning (09/10/2025)
Planejamento da **primeira entrega funcional do sistema**, focando no módulo de **Cadastro & Acesso**. Foram definidas as User Stories US-001 a US-006, incluindo separação entre dados cadastrais, responsáveis e ficha médica, além da inclusão de perfil neurodivergente. Foram estabelecidos requisitos não funcionais relacionados a LGPD, criptografia e testabilidade, além de regras de negócio para alertas de condições médicas críticas e privacidade de dados por tutela. A ordem de implementação foi definida via método MoSCoW.

### Sprint 2 Review (13/10/2025)
Validação do **fluxo completo de cadastro e autenticação** com demonstração do cadastro completo de aluno, cadastro de responsáveis, registro de ficha médica e fluxo de autenticação e perfis. O cliente aprovou as regras de privacidade, confirmou os alertas médicos e validou os requisitos não funcionais aplicados. Como feedback, foram solicitados ajustes no layout de formulários e melhorias na visualização dos dados do aluno. Os próximos passos incluíram o início do desenvolvimento do módulo de Presença e Justificativas.
